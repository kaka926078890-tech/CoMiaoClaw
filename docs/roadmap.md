# 未来内容规划

在最小 MVP（本地模型 + 单会话聊天）之后，按优先级规划的后续功能与方向。

---

## P0：核心能力

### 跨对话记忆

- **说明**：模型本身无持久记忆，通过外部存储 + 每次请求注入相关记忆实现「跨对话记忆」。
- **要点**：为会话/用户维护记忆条目（时间、内容、可选摘要）；每次调用前检索或选取最近/最相关条目，拼成一段文本注入 system 或首条 user；注意上下文长度，可做条数或 token 上限。
- **可选**：向量检索（embedding + 相似度）做相关性筛选；约定「何时写记忆」（如每轮结束总结要点写入）。

### 主 agent + 子 agent 协同

- **说明**：主 agent 将复杂任务拆成子任务，派发给「子 agent」执行，结果回传后主 agent 综合并继续。
- **要点**：子 agent 可视为「同模型、不同 system/目标的又一次调用」。主 agent 通过约定输出格式（如 `DELEGATE: 子任务描述 | 子角色`）或 tool（如 `delegate_subtask(description, role)`）声明派发；后端解析后构造子任务请求、调模型、将回复作为新消息或 tool 结果塞回主会话。
- **扩展**：支持多子任务并行、子任务超时、不同模型担任子 agent。

## P1：工具与多会话

### 技能/工具

- **说明**：模型能调用外部能力（读文件、执行命令、搜网页等）。
- **要点**：定义少量 tools（名称、描述、参数 schema）；在请求中通过 Ollama 的 tools 能力或自定义格式（如模型输出 `TOOL_CALL: name(args)`，后端解析执行后拼回 messages）。先实现 1～2 个简单工具验证流程。
- **扩展**：与主/子 agent 结合，子 agent 专司某类工具。

### 多会话 / 会话管理

- **说明**：支持多个独立会话，按会话 ID 隔离上下文与记忆。
- **要点**：存储层按 session_id 维护 messages 与记忆；网关/前端携带 session_id；可选会话列表与切换 UI。
- **扩展**：会话持久化、导入导出、归档与清理策略。

## P2：体验与扩展

### 多端 / 多 channel

- **说明**：除 Web 聊天页外，对接其他入口（如 Telegram、微信、Slack）。
- **要点**：各 channel 作为「入站消息源」，统一转成内部消息格式投递到网关；出站时根据 channel 回写。可参考 OpenClaw 的 channel 抽象。
- **扩展**：不同 channel 的权限、限流、会话隔离策略。

### Compaction / 摘要

- **说明**：当对话历史过长时，对旧消息做摘要或压缩，保留关键信息，避免超出上下文窗口。
- **要点**：设定长度或轮数阈值；触发时用模型生成摘要或抽取要点，替换或追加为一条「历史摘要」消息；可结合记忆模块写入长期记忆。
- **扩展**：分块摘要、按主题聚合、与跨对话记忆联动。

### 多模型与 fallback

- **说明**：支持配置多个模型（如本地 7B + 云端大模型），主模型不可用或超时时自动 fallback。
- **要点**：在模型层或网关层维护 provider 列表与优先级；请求失败或超时时切换下一候选；统一请求/响应格式以便多模型透明切换。
- **扩展**：按任务类型路由（如简单问答用 7B，复杂推理用大模型）、成本与延迟策略。

---

## 优先级说明

- **P0**：与「像 claw 一样可用」直接相关，建议在 MVP 稳定后优先做。
- **P1**：显著提升可用性与场景覆盖，在 P0 落地后安排。
- **P2**：体验优化与规模扩展，可按需求与资源分批实施。

具体任务可拆成 issue 或迭代计划，每项标注对应优先级与依赖关系。
