# find-skill

本技能只做一件事：**教你怎么根据用户意图决定「要不要用技能、用哪个技能」**。不在这里维护技能名单，名单由系统从 workspace/skills 自动扫描并注入到你的上下文中（即「可用技能」那一块）。

## 原则

1. **技能列表从哪来**：系统已在你的上下文中注入了「**可用技能**」列表，每行格式为「**技能名**：名称 + 简短描述」。该列表随 workspace/skills 里增删技能自动变化，无需在 find-skill 里写死。
2. **何时用技能**：当用户请求**有可能被某条技能的描述所覆盖**（即「用这个技能能完成或更好完成」）时，就应使用技能。
3. **怎么用**：先根据「可用技能」里的**描述**判断哪条与当前意图最匹配，在回复中写 `SKILL: <技能名>`（技能名即该条里的目录名），系统会加载该技能 SKILL.md 全文；加载后按该技能全文的说明执行（例如若要求写 `FETCH_URL: ...` 就再写）。

## 步骤（每次收到用户消息时）

1. 看上下文中「可用技能」列表：每条有**技能名**和**描述**。
2. 判断：当前用户意图是否「可以通过其中某条技能完成」？若可以，且用技能比你自己硬答更合适，则选描述最匹配的那条。
3. 若选中某技能：回复里先写 `SKILL: <技能名>`，加载后按该技能说明做（如写 FETCH_URL、按步骤操作等）。
4. 若没有匹配的技能：直接用自己的能力回答即可。
5. 若不确定是否该用技能、或有多条可能匹配：可先写 `SKILL: find-skill` 加载本技能，本技能会再次强调上述原则，然后你根据「可用技能」列表再选一次。

## 示例（仅说明逻辑，不穷举技能）

- 用户问「现在几点」→ 看列表里有没有描述与「时间、日期、当前时间」相关的技能 → 有则 `SKILL: 该技能名`，再按该技能说明写 FETCH_URL 等。
- 用户给了一个链接要内容 → 看列表里有没有描述与「抓取网页、链接内容、URL」相关的技能 → 有则加载该技能并按说明写 FETCH_URL。
- 用户问一般知识、不需要联网或工具 → 列表里没有明显匹配的技能 → 直接回答，不必写 SKILL。

注意：有 DELEGATE 时勿混用 SKILL；一次可写多行 `SKILL: a`、`SKILL: b` 加载多个技能。
