# 对话总结 2026-03-03（11）

## 需求与结论

本对话围绕「支持远程 API 调用（如 DeepSeek）」的方案讨论与记录，未改业务代码，仅产出计划文档与总结。

1. **远程 API 可行性**  
   确认网关可通过统一 LLM 层（llm.ts）+ OpenAI 兼容适配（openai-compat.ts）支持远程模型；配置 `LLM_PROVIDER`、`OPENAI_API_KEY`、`OPENAI_BASE_URL`、`OPENAI_MODEL` 即可。

2. **远程走 tool call、本地保留商定协议**  
   远程路径使用 API 的 function calling：请求带 `tools`，模型返回 `tool_calls`，网关执行后追加 tool result 再调；不在回复正文解析 DELEGATE/TIME_TASK/SKILL/FETCH_URL。Ollama 路径保留现有 AGENTS.md 与正文协议解析，不改动。

3. **人设双文件与内容共用**  
   AGENTS.md 继续供 Ollama 使用，内容保留。新建 AGENTS-TOOLS.md 供远程使用，写「身份 + 通用行为」与「何时用哪个工具」；与 AGENTS 共用身份、当前时间、综合子任务等段落（共用片段或两文件相同开头）。config 增加 agentsPathTools，bootstrap 按 provider 选 readAgents() 或 readAgentsTools()。

4. **前端切换**  
   页面增加「Ollama / 远程 API」切换按钮；GET /config 返回 llmProvider；请求体支持 provider；前端 state selectedProvider 并随请求下发。

5. **计划文档**  
   已生成 `docs/remote-api-and-tool-call-plan.md`，记录完整实现方案（配置、llm 层、openai-compat、人设双文件、bootstrap 分支、主对话分流、前端切换、验收要点），便于后续按步骤落地。
