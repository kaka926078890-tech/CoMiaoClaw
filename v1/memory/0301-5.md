## 本对话涉及的需求与结论

### P0 主 agent + 子 agent 协同规划

- **需求确认**：主 agent **自主**决定何时派发子任务（用户正常提问、无需命令词）；子结果回传后由主 agent **再综合并继续**，产出最终回复。
- **派发方式**：主 agent 在回复中输出约定格式 `DELEGATE: 子任务描述 | 子角色名`，网关解析后调子 agent（同模型、不同 system），子结果塞回主会话后再调主 agent 得综合回复。
- **完整版计划文档**：已生成 [docs/p0-main-sub-agent-plan.md](../docs/p0-main-sub-agent-plan.md)，包含端到端流程、阶段一～六（约定与解析、子角色配置、子 agent 调用、非流式/流式接入、persona、前端可选）、以及 8 项任务清单。
