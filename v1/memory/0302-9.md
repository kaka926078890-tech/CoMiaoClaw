# 对话总结 0302-9

## 本段对话主要改动与结论

### 浏览器与代理
- **Headless 浏览器**：网关新增 Playwright 能力，协议 `BROWSER_NAVIGATE: <url>`，技能 `gateway/data/skills/browser/SKILL.md`；`gateway/src/browser.ts`（navigateAndSnapshot）、`gateway/src/browser-protocol.ts`（解析/剥离）。
- **代理**：访问外网（如 docs.openclaw.ai）需代理时，Playwright 读取 `HTTP_PROXY`/`HTTPS_PROXY`；已在 `browser.ts` 中按环境变量注入 context 代理。根目录 `.env` 已配置 `HTTP_PROXY=http://127.0.0.1:7897`、`HTTPS_PROXY=...`（端口 7897）；`.env.example` 增加可选说明。

### 时间与配置
- **外部时间**：默认使用 WorldTimeAPI（`USE_EXTERNAL_TIME` 默认 true），仅当显式 `USE_EXTERNAL_TIME=0` 时用本机时间；避免服务器时钟分钟不准。bootstrap 中外部时间标签为「当前日期时间（网络）」。
- **DeepSeek R1 拒答时间**：在 AGENTS 与 bootstrap 时间块中明确「供你直接使用并转述」「不属超权」「禁止以超出权限拒绝」，减少模型误判。

### 人设与技能/子角色
- **格式分离、不可混用**：AGENTS 明确「子角色用法」固定为 `DELEGATE: 任务 | 子角色名`（子角色名仅来自「当前子角色及其能力」）；「技能用法」固定为 `SKILL: 技能名` + 该技能要求的协议行（如 FETCH_URL、BROWSER_NAVIGATE）。不得把技能名写在 DELEGATE 的「子角色名」位置。
- **规则**：新增 `.cursor/rules/agents-persona.mdc`，约定修改 AGENTS/人设时避免硬编码、写死、过度限定；project.mdc 已引用。
- **派发指令保留显示**：流式分支中 `main_reply_clean` 下发完整主回复（含 DELEGATE 行），便于测试与排查；session 仍存完整主回复。

### 流式超时
- **问题**：DeepSeek R1 长时间思考导致固定 120s 总超时触发，但流实际仍在继续。
- **修复**：改为「空闲超时」180s；每次 `onThinking`/`onChunk` 重置计时器，仅当连续 3 分钟无任何数据时判定超时。

### 技能与前端
- 现有技能：browser、web-fetch、url-scraper、current-time、find-skill、agent-commons 等；find-skill 按「可用技能列表描述」匹配意图，不维护硬编码表。
- 前端：可展示 skill_loaded、fetch_url_done、browser_navigate_done 等 SSE 事件；主回复区在派发时保留 DELEGATE 指令显示。

## 涉及文件（本段对话相关）

- `gateway/src/index.ts`：流式空闲超时、main_reply_clean 完整主回复、BROWSER_NAVIGATE 注入

- `gateway/src/browser-protocol.ts`：parseBrowserNavigateUrls、stripBrowserNavigateFromReply

- `gateway/src/bootstrap.ts`：外部时间标签「网络」、loadBootstrap(overrides)
- `gateway/data/AGENTS.md`：子角色/技能格式固定、不可混用
- `.cursor/rules/agents-persona.mdc`：新增；`.cursor/rules/project.mdc`：引用
- `.env.example`：USE_EXTERNAL_TIME、HTTP_PROXY/HTTPS_PROXY 说明
- `gateway/data/skills/browser/SKILL.md`、current-time、find-skill、agent-commons 等
