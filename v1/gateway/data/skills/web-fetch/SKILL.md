# web-fetch

当需要最新信息或访问某链接时使用本技能，弥补本地模型不联网、数据旧的问题。

## 使用步骤

1. 根据用户问题或上下文确定需要访问的 URL（必须是 http 或 https）。
2. 在你的回复中单独写一行：`FETCH_URL: <url>`（将 `<url>` 替换为完整地址）。
3. 系统会抓取该页面并将内容注入对话，你再根据注入的网页内容回答用户。

## 示例

用户问「今天 GitHub 首页有什么重要公告？」时，你可以回复：

```
FETCH_URL: https://github.com
```

系统抓取后会把页面文本注入，你再基于内容总结或回答。

注意：一次回复可写多行 `FETCH_URL: <url>` 抓取多个页面；有 DELEGATE 时勿混用本技能。
