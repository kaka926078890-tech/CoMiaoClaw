# 对话总结 · 2026-03-01（4）

## 本对话涉及的需求与结论

### 环境与配置
- **本地安装启动**：gateway + frontend 分别 `npm install` 与 `npm run dev`；网关 3000、前端 5173；需本机 Ollama 与模型（如 deepseek-r1:8b）。
- **单一配置源**：项目根目录 `.env` 配置 `OLLAMA_MODEL` 等；网关用 dotenv 从根目录加载，暴露 `GET /config`；前端通过 `getGatewayConfig()` 拉取并缓存，不再使用 `VITE_OLLAMA_MODEL`。
- **一句命令启动**：根目录新增 `package.json`，使用 `concurrently`，执行 `npm run dev` 或 `npm start` 同时启动网关与前端（跨平台）。`script/start.sh` 等仍可在有 bash 的环境使用；Windows 无 bash 时用根目录 `npm run dev`。
- **验证规则**：`.cursor/rules/project.mdc` 增加「验证与重启」：涉及网关/前端验证时须用 script 命令重启后再验证；日志在 `script/logs/`。

### 跨对话记忆（已实现）
- **配置**：`gateway/src/config.ts` 增加 `memoryPath`、`personaPath`、`memoryInjectCount`、`memoryInjectMaxChars`、`memoryEntryMaxChars`（可环境变量）。
- **模块**：`gateway/src/memory.ts`（`loadMemory`、`appendMemory`、`getMemoryRaw`）、`gateway/src/persona.ts`（`loadPersona`）；`gateway/data/` 目录，`persona.md.example`；`.gitignore` 排除 `gateway/data/memory.md`。
- **接入**：`gateway/src/index.ts` 在 `/chat` 与 `/chat/stream` 中调用前注入人设+记忆，成功后写记忆；新增 `GET /memory`、`POST /session/clear`。
- **前端**：新启对话按钮调用 `clearSession()` + 清空本地消息；历史记录改为右侧抽屉，内嵌 `MemoryHistory` 展示 memory.md。

### 前端整体重新设计与完成
- **设计规范**：`index.css` 中 CSS 变量（颜色、圆角、阴影）；弱化明显边界线，用阴影替代边框（`--shadow-sidebar`、`--shadow-header`、`--shadow-input`、`--shadow-panel`、`--shadow-bubble` 等）。
- **布局**：左侧边栏（260px，可收起）：品牌、新对话、历史记录（打开右侧抽屉）、对话列表占位、底部用户区；主区：顶栏、错误条、消息区/空状态、底部输入；历史记录为右侧抽屉（最大 400px），遮罩关闭。
- **技术**：Tailwind CSS + `lucide-react` 图标（MessageCircle、PanelLeft、Plus、History、X、Menu、User、Bot、Paperclip、ArrowUp、Loader2、RefreshCw 等）；消息气泡带头像与阴影；输入区圆角+阴影，聚焦时 `shadow-input-focus`；加载态用 `Loader2` +「正在回复…」。
