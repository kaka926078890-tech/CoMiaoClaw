# 对话总结 · 2026-02-28（1）

## 项目定位

- **claw**：自建「类 OpenClaw」的本地 AI 助手，优先使用本地模型（如 Ollama + qwen2.5-coder:7b），不依赖 OpenClaw 成品。
- 参考 OpenClaw 的架构思路（gateway、agent、provider、前端），按 MVP 逐步实现。

---

## 已完成的改动与约定

### 文档与规则（项目根）

- **docs/minimal-mvp.md**：最小 MVP 说明。范围：模型层（Ollama `/api/chat`）、单会话、网关 `POST /chat`、简单前端；不包含记忆、主/子 agent、多 channel、工具。
- **docs/roadmap.md**：未来规划。P0 跨对话记忆与主/子 agent；P1 技能/工具与多会话；P2 多端、compaction、多模型 fallback。
- **.cursor/rules/project.mdc** 中约定：
  - docs 下文档名统一小写（kebab-case）。
  - 每次回复先「你好，克喵。」再正文。
  - 工程按可维护方式组织；全栈习惯、关注点分离、类型先行。
  - **总结命令**：用户发「总结」时，在 `memory/` 生成 `MMDD-N.md` 总结当前对话；N 为全局递增序号。

### 前端（frontend/）

- **技术栈**：Vite + React 18 + TypeScript，CSS Modules。
- **结构**：`src/api/gateway.ts`（POST /chat）、`src/config/env.ts`（网关 URL）、`src/types/chat.ts`、`src/hooks/useChat.ts`、`src/components/`（ChatConsole、MessageList、MessageInput、MessageBubble）。
- **行为**：单会话聊天页，输入发送、展示 user/assistant 消息，加载态与错误提示；开发时 Vite 将 `/chat` 代理到 `http://localhost:3000`。
- **运行**：`cd frontend && npm install && npm run dev`，需后端在 3000 提供 `POST /chat`（body `{ "message" }`，response `{ "reply" }`）。

### 记忆文件夹（memory/）

- **用途**：存放每次对话的压缩上下文总结，便于新对话快速了解项目。
- **命名**：`MMDD-N.md`（月日 + 全局序号，序号跨天递增）。
- **README**：说明命名规则与使用方式。
