# 对话总结 · 2026-03-02（7）

## 本对话涉及的需求与结论

### 子 agent 流式与重复数据

- **问题 1**：子 agent 思考过程未呈流式展示。原因包括：子 agent 思考区默认折叠（看不到更新）、Ollama 仅部分模型支持 `message.thinking`、SSE 可能被缓冲。
- **问题 2**：EventStream 中大量重复数据。原因：网关对每次 thinking/content 片段都发送**整段已累积内容**，导致同一条长内容被重复下发。
- **数据格式通用性**：DELEGATE 行与 SSE 事件（main_reply_clean、summary、sub_thinking、sub_chunk、sub_done）为约定契约，换大模型只需在新适配器里映射为相同事件；thinking 是否流式取决于该模型/API 是否支持。

### 已做修改

1. **网关**
   - `delegate.ts`：`sub_thinking` 改为发送增量 `thinking: t`；`sub_chunk` 改为发送增量 `chunk: c`（不再发整段 content），消除重复下发的整段内容。
   - `ollama.ts`：流式解析同时兼容 `message.thinking` 与顶层 `thinking`（兼容 generate 等格式）。
   - `index.ts`：SSE 使用统一 `write()`，每次写后尝试 `res.flush()`，减少缓冲、便于子 agent 流式及时到达前端。

2. **前端**
   - `gateway.ts`：解析 `sub_chunk` 时读取 `data.chunk`（增量），调用 `onSubChunk(role, data.chunk)`。
   - `useChat.ts`：`onSubThinking`/`onSubChunk` 按增量累积：当前 role 的 thinking/content 加上本次 delta 再写回 state，实现真正的流式展示。
   - `MessageBubble.tsx`：子 agent 思考区默认展开（`subFolded[role] ?? false` → 改为有结论时默认折叠）；有结论时默认折叠思考区（`subFolded[sub.role] ?? !!sub.content?.trim()`）。

### 待办 / 后续

- 若换非 Ollama 后端，需在新适配器中将该厂商的「思考」流映射为 `sub_thinking` 事件（若该 API 支持）。
- 子 agent 并行（无依赖时）见 `docs/openclaw-delegate-questions.md`。
