## 本对话涉及的需求与结论

- **500 与网关**：项目启动后 POST /chat 报 500，因当时无网关实现。已新增 `gateway/`（Express，POST /chat、POST /chat/stream），对接本地 Ollama，错误时返回 503 并带说明。
- **模型名可配置**：Ollama 模型名（如 qwen2.5-coder:7b）由前端环境变量 `VITE_OLLAMA_MODEL` 配置，请求体带 `model` 传给网关；网关支持请求体中的 `model` 覆盖。
- **脚本**：`script/start.sh`、`script/stop.sh`、`script/restart.sh` 统一启停网关(3000) 与前端(5173)；PID 写 `script/claw.pids`，日志在 `script/logs/`。
- **Git**：已添加 `.gitignore`（node_modules、.env、dist、script 运行时等），并完成初始提交与推送到 GitHub（kaka926078890-tech/CoMiaoClaw）。
- **Cursor 规则**：从 GenUI/ai-web 拷贝并适配了 `git-commit.mdc`、`doc-naming.mdc`、`no-code-comments.mdc`、`error-corrections.mdc`，并扩展 `project.mdc`（新对话读规则、API 契约优先、Git 提交规范）。

## 流式输出与体验（已完成）

- **网关**：新增 `POST /chat/stream`，SSE 返回 `data: {"chunk":"..."}` 与 `data: [DONE]`；Ollama 流式解析兼容 `response` 与 `message.content`；首包发送 `: \n\n` 与 `X-Accel-Buffering: no`；120s 超时。
- **前端**：`sendMessageStreaming` 消费 SSE；`useChat` 改为流式，用 ref 累积内容、首包时补占位助手消息，避免 React 批更新导致内容丢失；回答结束后 `onReplyComplete` 聚焦输入框。
- **自动滚动**：使用 `use-stick-to-bottom`，`MessageList` 用 `<StickToBottom>` + `<StickToBottom.Content>` 包裹；`MessageInput` 通过 `forwardRef` 暴露 textarea，`ChatConsole` 在 onReplyComplete 时 focus。
- **问题修复**：助手气泡为空因 state 批更新与占位未就绪；已用 ref 累积 + 首包时再补占位 + onDone 时补占位修复。EventStream 空因首包未发与 Ollama 字段兼容；已加心跳、双字段解析与超时。

## 跨对话记忆（未实现，方案与文档已就绪）

- **结论**：跨对话记忆尚未开发；实现顺序定为「流式 + 自动滚动 + 聚焦」后再做。
- **方案对齐 OpenClaw**：记忆在部署环境用 **Markdown（.md）** 存储；人设用**各类文件**保存；网关每次调用模型前读人设 + 记忆，拼成一条 system 注入，每轮结束后向 memory.md **追加**一段。
- **Token 压缩**：写记忆用规则截断（单条字数上限）；读记忆设条数 + 总字符上限；人设保持简短；可选后续做「模型生成摘要」「历史摘要合并」、当前会话 Compaction（见 roadmap P2）。
- **实现文档**：已生成 **docs/cross-conversation-memory-implementation.md**，按步骤列出：配置与目录 → memory.ts（loadMemory/appendMemory）→ persona.ts（loadPersona）→ index.ts 接入读/注入与写记忆 → 验收。可按该文档逐步实现。
