# Figma 前端 UI 设计提示词（设计任务书）

交付给 Figma 的设计需求说明，用于设计 Claw 前端工程 UI。请据此产出可落地的界面与交互稿。**以下功能模块均需在设计中完整覆盖。**

---

## 一、产品与场景

- **产品**：Claw — 智能体对话控制台。用户通过 Web 前端与 AI 助手对话；助手可调用子 agent、技能、联网、文件等能力，对话过程存在多阶段（主回复、子任务执行、工具调用、综合回复等）。
- **用户**：需支持多会话、加载历史会话、查看/编辑工作区内的 Markdown 配置文件（编辑时需预览）、查看控制台日志与定时任务；对话中需看到调试向的中间过程（思考、子 agent、技能调用等）。

---

## 二、功能模块清单（必须全部包含）

### 2.1 主框架与导航

- 左侧可折叠侧栏，内含入口：**新对话**、**历史记录**、**工作区**、**控制台日志**、**定时任务**、**对话列表**（当前会话列表）；底部可预留用户/设置区。
- 主区域：当前会话的**聊天记录区** + 底部输入框。
- 顶部：侧栏折叠按钮、标题、**模型选择**、可选设置入口。

### 2.2 聊天记录区

- **消息列表**：按时间序展示当前会话的消息；每条包含角色（用户 / 助手）、内容、时间等。
- **输入框**：多行输入、发送按钮、支持流式回复时的加载态。
- **消息形态**：用户消息、助手正文、可选附件或引用；助手消息在「当前对话」中可含思考、子 agent 块、技能调用等调试信息（见 2.8），在「加载历史」时仅展示正文（见 2.7）。

### 2.3 工作区与配置文件

- **工作区面板**：独立面板或抽屉，以树形或列表展示工作区根文件与目录：
  - 根文件：AGENTS.md、SOUL.md、MEMORY.md、TOOLS.md、IDENTITY.md、USER.md 等；
  - 目录：agents/（子 agent 配置，如 researcher.md、coder.md）、skills/（各技能子目录，内含 SKILL.md）。
- **文件打开与编辑**：点击文件后在主区或弹窗中打开；对 **.md 文件必须提供编辑 + 预览**：
  - **Markdown 预览器**：分栏（左编辑右预览）或 Tab（编辑 / 预览切换），实时渲染，保证编辑任意 .md 时都有预览。
  - 保存、未保存提示、错误态。

### 2.4 技能

- **在工作区中的体现**：skills/ 下各技能目录及 SKILL.md 通过工作区文件树展示与编辑（见 2.3）；编辑 SKILL.md 时同样需要 Markdown 预览。
- **在对话中的体现**：当助手调用技能时，在聊天记录区展示**技能使用过程**（如「正在使用技能：xxx」「已注入」），用于调试；可与控制台日志联动（见 2.6）。

### 2.5 子 agent

- **在配置中的体现**：agents/ 下各子角色 .md 通过工作区展示与编辑（见 2.3）。
- **在对话中的体现**：当存在多个子 agent 时，在聊天记录区展示**子 agent 的并发/合并调用过程**（哪些在运行、开始/结束、按 agent 或时序分组）、各子 agent 的思考与结论块，与主回复、综合回复明确区分；可折叠/展开，便于调试（见 2.8）。

### 2.6 日志监控区（控制台日志）

- **控制台日志面板**：独立面板或抽屉，展示请求/响应、错误、工具调用等日志；与侧栏「控制台日志」入口对应。
- 内容可为流式或列表；支持按时间、类型筛选或搜索（若需要）；视觉上区分信息/警告/错误等级。
- 可与对话中的「技能使用过程」「子 agent 过程」在信息上联动（如点击可跳转或高亮对应请求）。

### 2.7 定时任务区

- **定时任务面板**：独立面板或抽屉，与侧栏「定时任务」入口对应。
- 功能：任务列表（名称、类型、间隔、启用状态、上次运行等）、新建任务、编辑、删除、启用/禁用。
- 新建/编辑时需表单：名称、类型（如心跳、写时间文件、agent 指令等）、间隔（分钟）、启用开关及类型相关字段（如指令内容）。

### 2.8 历史记录与记忆

- **历史会话**：侧栏「对话列表」展示历史会话（标题、时间），点击加载该会话的**聊天记录**。
- **加载历史时的展示**：只展示**正文**（用户消息 + 助手最终回复正文），不展示思考、子 agent 中间输出、技能调用过程等，便于快速回顾。
- **记忆文件**：后端记忆文件记录完整正文；前端历史视图仅展示「正文」；可预留「查看记忆」入口（若产品需要）。

### 2.9 对话过程中的「调试向」展示（仅当前对话）

以下在**当前对话进行时**展示，**加载历史时不展示**；需与正文明确区分，可折叠/收纳。

- **思考过程（Thinking）**：独立区域或可展开块，标签如「思考」，样式弱化（如灰色、次要字体）。
- **子 agent 过程**：并发/合并调用状态、各子 agent 的思考与结论块；可按 agent 或时间线展示，可折叠。
- **技能使用过程**：正在调用的技能名、状态（进行中/已注入）；可与控制台日志区联动。

## 三、设计产出建议

- **主界面**：布局（侧栏 + 主区 + 输入）、响应式或主要分辨率；侧栏各入口与对应面板的对应关系。
- **聊天记录区**：消息气泡/块；正文 vs 思考、子 agent 块、技能调用状态的视觉区分；可折叠/展开规则；历史仅正文的样式差异。
- **工作区**：文件树/列表 + 编辑 + **Markdown 预览** 双栏或 Tab；保存/未保存/错误态。
- **控制台日志区**：列表或流式展示、等级区分、与对话调试信息的关联方式。
- **定时任务区**：列表、表单（新建/编辑）、操作按钮与状态。
- **通用**：加载中、错误、空状态、未保存提示等组件与状态。

---

## 四、参考与约束

- 现有 V1 前端为 React + CSS Modules，可沿用组件化思路；Figma 产出需便于后续在 React 中实现。
- 设计风格：清晰、信息层级分明；调试信息可弱化但可发现、可展开。
- 无障碍：关键操作具备可聚焦、可读性，符合基本 a11y 要求。
